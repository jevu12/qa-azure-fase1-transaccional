⚡ Azure DevOps · QA Automation Skill

QA Azure Fase 1 Transaccional

Skill para ejecutar la Fase 1 del flujo QA en Azure DevOps: detección de artefactos, creación idempotente de QA Tasks, Test Plan, Test Cases, vínculos y trazabilidad auditable.

📦 Ver en GitHub 📖 Documentación

🚀 Instalación rápida

npx skills add https://github.com/jevu12/qa-azure-fase1-transaccional

✨ ¿Qué hace esta skill?

🔍

Detección inteligente

Lee artefactos existentes por User Story antes de crear, garantizando idempotencia y sin duplicados.

⚙️

Creación transaccional

Crea QA Tasks, Test Plan, Test Suite y Test Cases solo cuando faltan. Nunca elimina ni recrea artefactos completos.

🔗

Vínculos y trazabilidad

Vincula automáticamente US ↔ Task ↔ Test Case. Genera decisions_log auditable por cada acción.

📋

Pipeline State

Actualiza outputs/pipeline-state.json en cada etapa. Permite retomar ejecuciones interrumpidas.

🐛

Gestión de bugs

Asigna bugs al último responsable de la US distinto al QA autenticado. Deduplicación con umbral de confianza.

📸

Evidencias segmentadas

Almacena capturas bajo outputs/evidencias/<sprint>/US-X/TC-Y/ con manifest por caso de prueba.

🎛️ Modos de ejecución

Modo Descripción Estado
full-pipeline Detección + ingesta + análisis + diseño completo Disponible
analysis-only Solo detección + ingesta + análisis QA Disponible
design-only Solo detección + ingesta + diseño de Test Cases Disponible
execution-only Ejecución/reporter/evidencias (requiere autorización del Orquestador) Extensión compat
detection-only Solo detección y diagnóstico, sin mutaciones Disponible

🤖 Agentes especializados

01

Orquestador

Clasifica US, aplica reglas y delega a agentes.

02

Detector de Artefactos

Lee estado actual en ADO antes de toda creación.

03

Ingestor

Crea QA Tasks faltantes como hijas de cada US.

04

Análisis

Completa QA Tasks de análisis con contenido ISTQB.

05

Diseño

Crea/corrige Test Cases en Ready con Steps XML.

06

Ejecutor

Ejecuta TCs con Playwright y captura evidencias.

07

Reporter de Bugs

Crea bugs con ReproSteps HTML y asignación al dev.

08

Gestor de Evidencias

Inventaria, adjunta y comenta evidencias en ADO.

📐 Reglas obligatorias

  • ✅ Idempotencia primero: nunca crear sin detectar antes
  • 🚫 Anti-duplicado por título, patrón y relación de negocio
  • 🔒 Sin operaciones destructivas ni eliminación de artefactos
  • 🛑 Sin mutaciones si MCP no está validado
  • 🔑 PAT solo desde variable de entorno AZURE_DEVOPS_EXT_PAT
  • 📁 Evidencias segmentadas por US/TC, nunca a nivel US
  • ⛔ No cerrar run sin evidencia validada por paso (BLOCKED_EVIDENCE)
  • 👤 Identidad resuelta desde usuario autenticado MCP, sin IDs hardcodeados

💬 Prompt recomendado

Inicio rápido
Usa $qa-azure-fase1-transaccional para ejecutar Fase 1 en Azure DevOps.
Toma la configuración desde inputs/project-config.json, procesa el sprint actual
en modo full-pipeline, no dupliques artefactos y entrega un resumen auditable
por US con acciones CREATE/UPDATE/SKIP/BLOCK/ERROR.
